# Version 0.11b — 2026-07-23

## Исправления

### pdf_to_md.py
- **Чанкование финальной чистки:** документы >25K символов разбиваются на части по 20K, исключён обрыв контента (finish_reason=length)
- **Склейка таблиц:** удалена локальная `_merge_broken_tables()`, используется общая из utils.py
- **VLM-режим:** исправлена передача pipeline_opts в DocumentConverter
- **Промежуточные файлы:** копируются в tmp/ с датой вместо удаления
- **Config:** добавлено применение base_url из config.yaml к utils.PROVOD_URL
- **Неиспользуемые импорты:** удалены `import httpx` в mode_vlm()

### docx_to_md.py
- **Склейка таблиц:** добавлен импорт и вызов `merge_broken_tables()` из utils (шаг 4b)
- **Промежуточные файлы:** копируются в tmp/ с датой вместо удаления
- **Отладка:** добавлено логирование размера на каждом шаге
- **Сохранение raw:** через save_intermediate с backup

### utils.py
- **`merge_broken_tables()`:** новая общая функция склейки таблиц для всех конвертеров.
  Правило: если после таблицы идёт продолжение с `|` — даже через пустые строки,
  произвольный текст, маркеры «Продолжение», «Окончание», `<!-- -->` —
  всё считается одной таблицей. Повторные шапки удаляются.
  Останавливается на заголовке нового раздела (`#` или `##`).
- **Исправление `_is_new_section_header`:** заменён `startswith('##')` на `startswith('#')`,
  т.к. Docling SimplePipeline для DOCX генерирует заголовки с одной решёткой
- **Удалён опасный блок пропуска произвольного текста:** функция больше не удаляет
  параграфы между таблицами, ошибочно принимая их за «продолжение»
- **POSTPROCESS_PROMPT:** ужесточено требование сохранять весь контент;
  явно указано не удалять заголовки разделов вида 1.1.1, 2.3.5;
  пункт "Remove duplicate headers" заменён на точечное удаление page-артефактов

### index_chromaDB.py
- **Новое чанкование:** вместо MAX_CHARS=1500 — MAX_TOKENS=8000 с перекрытием OVERLAP_TOKENS=200
- **Логика:** каждый раздел (`##`) — один чанк целиком, каждая таблица — один чанк целиком.
  При превышении лимита — дробление с перекрытием (chunk_with_overlap)
- **Удалены:** `add_chunk()`, `pack_table_rows()`, `cross_section_range()`, `MAX_CHARS`
- **Добавлены:** `estimate_tokens()`, `chunk_with_overlap()`, `finalize_section_chunk()`
- **CLI:** новые параметры `--max-tokens`, `--overlap-tokens`
- **Импорт sys:** восстановлен, вызывал ошибку NameError
